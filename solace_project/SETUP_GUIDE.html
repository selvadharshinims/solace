What is Firebase and why do you need it?

Firebase is a service by Google that gives your app a real cloud database — no server to manage, no backend code to write. Think of it like a shared Google Sheet that your app reads and writes to instantly from any device.

Right now Solace works in Demo Mode — data is saved only in the browser you're using. If you open the app on a different phone, it won't see the same data. Firebase fixes that.

FeatureDemo Mode (now)With Firebase
Data savedBrowser only (localStorage)Cloud — permanent
Multi-device❌ Each device is separate✅ All devices in sync
Real-time sync❌ Manual refresh needed✅ Under 1 second
Admin → Residents❌ Same browser only✅ All phones instantly
CostFreeFree — 100 users, 1 GB
✅ Completely free for this use case Firebase Spark (free plan) supports up to 100 simultaneous users, 1 GB storage, 10 GB/month bandwidth — more than enough for any residential community.
1

Create a Firebase Account

  1. Open your browser and go to console.firebase.google.com
  2. Click "Sign in with Google" — use any Google/Gmail account you have
  3. You'll see the Firebase Console homepage — this is your control panel
💡 Tip Consider using a dedicated Gmail account for the community (like solacecommunity@gmail.com) rather than a personal account, so others can access it if needed.
2

Create a New Project

  1. Click "Create a project" on the Console homepage
  2. Enter a project name — e.g. solace-community or care-home-app
  3. Click Continue
  4. On the Google Analytics screen — turn it off (not needed) → click Create project
  5. Wait ~20 seconds for Firebase to set up your project
  6. Click Continue when you see "Your new project is ready"
💡 Project name tip Your project name becomes part of the database URL. Use lowercase letters and hyphens only — e.g. my-care-home not My Care Home.
3

Enable Realtime Database

Firebase has two database products. Solace uses Realtime Database — make sure you pick this one (not Firestore).

  1. In the left sidebar, click "Build" to expand it
  2. Click "Realtime Database"
  3. Click the blue "Create Database" button
  4. Choose a location closest to your users:
    · asia-southeast1 — India, South/Southeast Asia
    · us-central1 — United States
    · europe-west1 — Europe
  5. Click Next
  6. Choose "Start in test mode" for now
  7. Click Enable
⚠️ Test Mode Warning Test mode allows open read/write for 30 days. This is fine for now, but you must set proper security rules in Step 6 before sharing with residents.
📌 Note your database URL After enabling, you'll see a URL like: https://your-project-default-rtdb.asia-southeast1.firebasedatabase.app — copy this, you'll need it in Step 5.
4

Register a Web App & Get Config

  1. Click the ⚙️ gear icon (top-left, next to "Project Overview")
  2. Select "Project settings"
  3. Scroll to the "Your apps" section at the bottom of the page
  4. Click the </> Web icon
  5. Enter a nickname: Solace Web App → click "Register app"
  6. You'll see your firebaseConfig — it looks like this:
const firebaseConfig = {
  apiKey:            "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXX",
  authDomain:        "solace-community.firebaseapp.com",
  databaseURL:       "https://solace-community-default-rtdb.asia-southeast1.firebasedatabase.app",
  projectId:         "solace-community",
  storageBucket:     "solace-community.appspot.com",
  messagingSenderId: "123456789012",
  appId:             "1:123456789012:web:abcdefabcdef"
};

What each field means:

apiKey
Public API key that identifies your Firebase project. Safe to include in client code.
authDomain
Used for authentication flows. Usually YOUR_PROJECT.firebaseapp.com
databaseURL
Critical. The URL of your Realtime Database — must include the region suffix.
projectId
Your project's unique identifier used across Firebase services.
storageBucket
For file storage. Not used by Solace but include it anyway.
messagingSenderId
For push notifications. Not used by Solace but include it anyway.
appId
Unique ID for this specific registered web app.
📋 Copy the full config block — you'll paste it into firebase-config.js in the next step.
5

Paste Config into firebase-config.js

Open firebase-config.js in any text editor (Notepad, VS Code, Notepad++). Find the firebaseConfig block and replace each "YOUR_..." placeholder with your real values from Step 4.

// BEFORE — placeholders:
export const firebaseConfig = {
  apiKey:            "YOUR_API_KEY",
  authDomain:        "YOUR_PROJECT.firebaseapp.com",
  databaseURL:       "https://YOUR_PROJECT-default-rtdb.firebaseio.com",
  projectId:         "YOUR_PROJECT",
  storageBucket:     "YOUR_PROJECT.appspot.com",
  messagingSenderId: "YOUR_SENDER_ID",
  appId:             "YOUR_APP_ID"
};

// AFTER — your real values:
export const firebaseConfig = {
  apiKey:            "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXX",
  authDomain:        "solace-community.firebaseapp.com",
  databaseURL:       "https://solace-community-default-rtdb.asia-southeast1.firebasedatabase.app",
  projectId:         "solace-community",
  storageBucket:     "solace-community.appspot.com",
  messagingSenderId: "123456789012",
  appId:             "1:123456789012:web:abcdefabcdef"
};
⚠️ Common mistake — databaseURL region Your databaseURL must include the region suffix. If you chose asia-southeast1:
✅ Correct: https://your-project-default-rtdb.asia-southeast1.firebasedatabase.app
❌ Wrong: https://your-project-default-rtdb.firebaseio.com
💡 One file to edit, both apps update firebase-config.js is shared by both index.html and admin.html. You only edit one file and both apps connect to Firebase.

Save the file. Reload index.html — the "Demo Mode" notice will disappear once Firebase is detected and connected.

6

Set Security Rules

Security rules control who can read and write to your database. Set these before sharing the app with residents.

  1. In Firebase Console → Realtime Database → click the Rules tab
  2. Select all existing text and delete it
  3. Paste the rules below
  4. Click Publish
{
  "rules": {
    "apartments": {
      ".read": true,
      ".write": false,
      "$apt": { ".write": true }
    },
    "activities": {
      ".read": true,
      ".write": false
    },
    "notices": {
      ".read": true,
      ".write": false
    },
    "requests": {
      "$apt": { ".read": true, ".write": true }
    },
    "checkins": {
      "$apt": { ".read": true, ".write": true }
    },
    "moodLog": {
      "$apt": { ".read": true, ".write": true }
    },
    "chat": {
      "$apt": { ".read": true, ".write": true }
    }
  }
}
PathReadWriteWhy
/apartments✅ Anyone❌ Blocked globallyLogin page reads apartment list. Admin panel writes via its own credentials.
/activities✅ Anyone❌ BlockedAll residents see activities. Admin-only write.
/notices✅ Anyone❌ BlockedSame as activities.
/requests/$apt✅ Per apartment✅ Per apartmentResidents submit and read their own requests.
/checkins/$apt✅ Per apartment✅ Per apartmentResidents log their own check-ins.
/moodLog/$apt✅ Per apartment✅ Per apartmentResidents log their own mood.
/chat/$apt✅ Per apartment✅ Per apartmentFlat-level chat, scoped to each apartment.
7

Add Apartments via Admin Panel

Once Firebase is connected, manage all apartments directly from the admin panel — no Firebase Console editing needed.

  1. Open admin.html in your browser
  2. Log in: username admin · password admin123
  3. Click Residents in the sidebar
  4. Click "+ Add Apartment"
  5. Fill in Apartment number (e.g. A-305), Passcode (e.g. 5678), Resident name, email, emergency contacts
  6. Click Save Apartment — written to Firebase instantly
  7. Repeat for all apartments
💡 Emergency contacts format Enter contacts like this: Son:9876543210, Daughter:8765432109
Separate multiple contacts with a comma. Format is Name:PhoneNumber
✅ Residents can log in immediately As soon as you save an apartment, the resident can open index.html, pick their flat from the dropdown, and enter their passcode to log in.
8

Host Your Files (Optional but Recommended)

To let residents open the app from their own phones, host the files online. All options below are free.

OptionCostTimeBest For
Netlify DropFree30 secondsQuickest — drag and drop your folder
Firebase HostingFree5 minSame account, auto HTTPS, custom domain
GitHub PagesFree10 minGood if you use GitHub
VS Code Live ServerFree1 minLocal WiFi testing only

Netlify Drop (easiest):

  1. Go to app.netlify.com/drop
  2. Drag your entire Solace project folder onto the page
  3. Netlify gives you a live URL like https://bright-star-12345.netlify.app
  4. Share that URL with residents — they can add it to their phone's home screen
⚠️ Voice assistant requires HTTPS The microphone / voice assistant only works on HTTPS or localhost. All hosting options above include HTTPS automatically. Opening via file:// from your computer will not work for voice.

Database Structure Reference

How your data is organized in Firebase. Useful for debugging or manual edits in the Firebase Console.

your-project-db/ ├── apartments/ │ └── A-305/ │ ├── passcode: "5678" │ ├── name: "selvadharshini" │ ├── email: "selvadharshinims@gmail.com" │ └── contacts/ │ ├── 0: {name: "son", phone: "8056116846"} │ └── 1: {name: "daughter", phone: "6379311304"} ├── activities/ │ └── {auto-key}: {time, name, location} ├── notices/ │ └── {auto-key}: {tag, title, body, date} ├── requests/ │ └── A-305/ │ └── {auto-key}: {type, notes, date, status, apt, name} ├── checkins/ │ └── A-305/ │ └── {auto-key}: {date, mood, apt, name} ├── moodLog/ │ └── A-305/ │ └── {auto-key}: {date, emoji, label, note} └── chat/ └── A-305/ └── {auto-key}: {text, sender, apt, time}

Troubleshooting

ProblemLikely CauseFix

Login: "Apartment not found"No apartments in Firebase yetAdd apartments via Admin → Residents page after connecting Firebase
Database URL error in browser consoleWrong databaseURLCopy the exact URL from Firebase Console → Realtime Database page (includes region)
Permission denied errorsSecurity rules blocking writeRe-paste the rules from Step 6 and click Publish again
Voice assistant doesn't workNeeds HTTPSHost the files online (Step 8). Voice won't work on file://
